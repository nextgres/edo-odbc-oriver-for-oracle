EDO extra connection attributes

To be able to alter the EDO specific features after making a connection, there are a number of EDO driver specific attributes you can use in the calls to SQLGetConnectionAttr (to read them) or to SQLSetConnectionAttr (to set them). These function can be called with the values as listed in the table below in the second parameter to this function (Attribute).

These attributes are enclosed in the installation package of the EDO driver in the file "EDO_Attributes.h" for use in your own programs.

Attribute

Description

SQL_ATTR_EDO_ENSURE_STATISTICS

SQLUINTEGER value 0 to disable the ensure statistics mechanism or 1 to enable it.

SQL_ATTR_EDO_PREFETCH_COUNT

SQLUINTEGER value. Number of rows to prefetch in the driver. The driver will use up to the amount of prefetch memory or the amount of prefetch rows, whichever boundary comes first.

SQL_ATTR_EDO_PREFETCH_MEMORY

SQLUINTEGER value. Number of bytes of cache memory to use in the driver while prefetching rows. The driver will use up to the amount of prefetch memory or the amount of prefetch rows, whichever boundary comes first.

SQL_ATTR_EDO_FAILOVER_COUNT

SQLUINTEGER value. Number of times to retry a connection to an Oracle RAC cluster after the program loses its connection.

SQL_ATTR_EDO_FAILOVER_TIME

SQLUINTEGER value. Number of seconds between retries to an Oracle RAC cluster after the program loses its connection.

SQL_ATTR_EDO_TRACE

SQLUINTEGER value. 0 to disable the EDO tracing mechanism or 1 to enable it.

SQL_ATTR_EDO_TRACEFILE

SQLCHAR* string. Name of the logfile to use for the EDO tracing mechanism.

SQL_ATTR_EDO_INTEGERSPEEDUP

SQLUINTEGER value. 0 to disable the integer speedup mechanism or 1 to enable it. This can interpret NUMBER's with 0 precision and 0 scale as binary integers.
