Connection attributes

Datasources have four primary attributes (DSN, UID, PWD, SERVER) that account for the connection to the database. Besides these four primary attributes there are a number of other attributes that serve to change the behaviour of the connection. The table below list all attributes in the attribute string as well as in the MS-Windows registry.

Connection strings

Connection strings can be used in calls like SQLConnect, SQLDriverConnect and SQLBrowseConnect. More than one attribute can be strung together in the connection string to one of these three functions. They use following format:

"<attribute>=<value>;..."

So a standard connection can be something like:

"DSN=ORCL;UID=scott;PWD=tiger;SERVER=ORCL1"

Registry attributes

Registry attributes are written in the MS-Windows registry under the "HKEY_LOCAL_MACHINE\SOFTWARE\ODBC\ODBC_INI\<datasource>" for system wide datasourcenames, and under "HKEY_CURRENT_USER\SOFTWARE\ODBC\ODBC_INI\<datasource>" for user datasources. These values can be managed with the ODBC manager and the datasource setup dialog of EDO.

Precedence

What happens if an attribute is set in the registry and has a different value in the connection string? Well, all the attributes in the registry are read in first, and after that the attribute values of the connection string are parsed. So the later (attributes from the connection string) take precedence over those from the registry. An application programmer can in this way always override local settings.

Explanation

DSN

DSN

Datasource name. This is the name application use to create a connection.

UID

UID

User ID. The login name of the user logging in on the datasource.

PWD

PWD

Password. Authentication of the user.

SERVER

ServerName

Oracle server. In effect the TNS connection string name.

 

Driver

Full path to the EDO driver DLL

RDO

ReadOnly

Readonly access mode. 0=read/write mode 1=readonly mode

Async enabled with multithreaded support (1) or async disabled (0) with no multithreading.

Autocommit on(1) or off(0).

Connection timeout in seconds

MID

MetaDataID

Metadata ID 1=catalog functions need exact identifier, 0=catalog functions accept search patterns like 'AB%' or 'INVOICE___'

OdbcCursor

Cursor support of the driver. Option 2 guarantees the highest speed and is default.

0 = Use cursor library if needed

1 = Use ODBC cursor library

2 = Use Driver/RDBMS cursor library

Packet size (does nothing for now)

Quiet mode. Driver does never display any dialog boxes or messages

Trace file for EDO driver trace if TraceLevel=1

Translate library. Name of the DLL containing SQLDriverToDatasource and vice versa.

Transaction isolation. (does nothing for now)

STA

StatisticsEnsure

Statistics ensure. Enables (1) or disables (0) the SQL_ENSURE option of the SQLStatistics call. disabling allows for higher performance in all situations.

PFC

PrefetchCount

Prefetch count. Number of records to prefetch and cache at once.

PFM

PrefetchMemory

Prefetch memory. Number of bytes for prefetch cache memory

FOC

FailoverCount

Failover count. Number of times to retry connection to Oracle RAC cluster.

FOT

FailoverTime

Failover time. Number of seconds between failover retry to Oracle RAC cluster.

MCL

ColumnMaxLength

Max column length for static binds to long columns.

ISU

IntegerSpeedUp

Integer speedup. If on (1) Oracle NUMBER with 0 precision and 0 scale are treated as binary integetrs. If off (0) Oracle NUMBER's are treated as DECIMAL's.

 

NLS_TYPE

Type of National Language Support (NLS)

0 = Use standard Oracle NLS

1 = Use standard ODBC settings for NLS

2 = Use specific user selected values for NLS

See the explanation about NLS settings of the datasource

 

NLS_DATE_FORMAT

String: Standard Oracle format for a date, as used in the TO_DATE function.

 

NLS_TIME_FORMAT

String: Standard Oracle format for a date, as used in the TO_TIMESTAMP function.

 

NLS_TIMESTAMP-_FORMAT

String: Standard Oracle format for a date, as used in the TO_TIMESTAMP function.

 

NLS_TIMESTAMP-_TZ_FORMAT

String: Standard Oracle format for a date, as used in the TO_TIMESTAMP function for use with datatypes that do support timezones.
