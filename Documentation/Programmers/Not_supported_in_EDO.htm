Not supported in EDO

Of course there is a catch. Like the fineprint in a contract there are some Oracle features that are not (yet) supported in version 1.0.0 of the EDO driver. The list in here has them all.

Feature

Description and why

Piecewise get of a LONG

All other datatypes are supported true SQLGetData without the cumbersome OCI interface to piecewise extraction of long datatypes. This simplifies the driver greatly. And CLOB and NCLOB are the new way of inserting and extracting long text data anyway. So this is not a great sacrifice.

SQLBulkOperations

A not heavily used function to manipulate rowsets in static arrays. Scheduled for the next release (1.1.0)

SQLSetPos

A not heavily used function to manipulate rowsets in static arrays. Scheduled for the next release (1.1.0)

Datatype BFILE

The ODBC standard has no way of dealing with the BFILE datatype of Oracle. One could argue that an extension to the ODBC standard would be in place. But not in this EDO release of yet.

Datatype UROWID

The ODBC standard has no way of dealing with the UROWID datatype of Oracle. One could argue that an extension to the ODBC standard would be in place. But not in this EDO release of yet.

Object datatypes and REF's

User defined datatypes (composites of the basic datatypes) and references to them are not supported in any way by the ODBC standard.

True UTF-16 Unicode

All functions in the EDO driver are tested against 8 bit databases (WE8ISO8859P15) only.
