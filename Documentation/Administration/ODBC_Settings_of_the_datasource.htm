ODBC Settings of the datasource

The first tab with information about the ODBC datasource is called "ODBC". This tab contains various settings

 

Field name

Description

Read only connection

Select to define a read only datasource. Only SELECT statements can be used on such a datasource. Very safe for production databases!

Ensure statistics info

Enables the SQL_ENSURE option of the SQLStatistics function. This ODBC function requests the statistics info of indexes and table columns. Do not select this if you want optimum performance. The ensure option can take a long time to complete depending on the number of rows in the table or index.

Set MetaDataID to TRUE

Select to set the default MetaDataID to true. This means that the catalog functions like SQLTables, SQLColumns, SQLPrimaryKeys, SQLForeignKeys, SQLStatistics, SQLTablePrivileges, SQLColumnPrivileges and SQLSpecialColumns will interpret the search values as string literals without search patterns. This enables for a slightly higher performance.

When not selected, the search values will be interpreted as search patterns, allowing for more flexibility in programs to search for metadata objects.

Use integer speedup

Select this option if you are using an application that generates true INTEGERS, and DECIMALS with distinct precisions and scales. This will mean that Oracle NUMBER's with no precision and scale will always be treated as binary integers (32 bits). This option can speed up your datasource considerably.

Maximum length of columns

Sets the maximum length of statically bound columns containing a long datatype like LONG, CLOB, NCLOB, BLOB or LONG RAW. These datatypes allow for 2GB of data and you might not have that much memory at your disposal.
