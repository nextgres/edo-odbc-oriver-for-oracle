Oracle settings of the datasource

The second tab with information about the datasource is called "Oracle". This tab contains various settings for the Oracle engine.

 

Field name

Description

Use Oracle prefetching

You can speed up your ODBC connection by setting an amount of memory apart for caching of the result set in the Oracle layer. If you wish to do so, you must enter the next two parameters.

Prefetch row count

Maximum number of rows to cache between the ODBC driver and the database. The driver will use up to the amount of prefetch memory or the amount of prefetch rows, whichever boundary comes first.

Prefetch memory

Maximum amount of bytes of memory cache to use between the ODBC driver and the database. The driver will use up to the amount of prefetch memory or the amount of prefetch rows, whichever boundary comes first.

Use Oracle failover support

Enable this option for use of the datasource on Oracle RAC failover clusters. If you wish to do so, you must enter the next two parameters.

Failover retry count

Number of times to retry a connection to the Oracle cluster

Failover retry timeout

Number of seconds to wait in between a retry attempt.
