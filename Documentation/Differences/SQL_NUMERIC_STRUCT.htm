SQL_NUMERIC_STRUCT

To accommodate the ISO datatype DECIMAL with a maximum of 32 digits precision and scale, the ODBC standard introduced a new datatype in version 3.x. This datatype is named SQL_NUMERIC_STRUCT and is defined as follows:

#define SQL_MAX_NUMERIC_LEN 16

typedef struct tagSQL_NUMERIC_STRUCT

{

SQLCHAR   precision;

SQLSCHAR scale;

SQLCHAR   sign; /* 1 if positive, 0 if negative */

SQLCHAR   val[SQL_MAX_NUMERIC_LEN];

}

SQL_NUMERIC_STRUCT;

 

This structure allows for storage of the precision and scale and the sign of the number. The actual mantissa of the number is stored in the 'val' part of the structure. It's stored with a radix of 256, so no bits of information are wasted. The 'precision' member holds the number of valid bytes in the 'val' member. This allows for the maximum number of: 3.4028236692093846346337460743177e+38

 

The official Oracle driver does not support this datatype. Binding a column to this datatype will yield strange and unpredicted results.

The EDO ODBC driver uses the SQL_NUMERIC_STRUCT as the primary way to pass on NUMBERS from the database. That is, if the numbers are not bound to other approximate number types such as SQL_FLOAT, SQL_REAL, SQL_DOUBLE or exact number types like SQL_SHORT, SQL_LONG or SQL_SBIGINT. How this is done, is described in the 'binding to a number' page.
