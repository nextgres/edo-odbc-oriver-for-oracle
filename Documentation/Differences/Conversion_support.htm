Conversion support (CAST and CONVERT)

There are two standard ODBC ways of data conversion: CAST and CONVERT.

CAST

Oracle: EDO:

The cast operator is the newer way to convert data values to another datatype. The cast operator does not work with the standard ODBC escape syntax. So there is no "{ CAST" syntax construction. Instead you can query the "SQLGetInfo" function for the SQL_SQL_92_EXPRESSIONS type. If this type returns the SQL_SVE_CAST bit as set, the RDBMS supports the following syntax

CAST ( value-exp AS data-type)

Whereas the data-type are de native RDBMS datatypes. This means of course that there is a drawback: you cannot write real portable SQL programs. But there is a second drawback as well. Oracle does support the CAST operator, but does **not** return the SQL_SVE_CAST bit. So there is no way of automatically determining the support for this operator by a real ODBC program.

 

CONVERT

Oracle: EDO:

The older and portable way of conversion is the "CONVERT" function. It   is a way to transpose data from one datatype to another. It is used if no standard casting method exist for the RDBMS at hand. Basically you convert a value to another datatype. Everything is done in the standard ODBC escape syntax. A basic conversion to another datatype is described as:

{ fn CONVERT( <value> , <datatype>) }

Which means that this is a valid way to convert a date to a string:

SELECT { fn CONVERT( { fn CURDATE() }, SQL_CHAR) } FROM DUAL;

The datatypes that you can use are similar to the ODBC macros, but there are some exceptions for the signed and unsigned variants. These are the datatypes that you can use in the CONVERT function.
