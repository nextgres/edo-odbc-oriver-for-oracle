RDBMS Implementation

The implementation of the database can be discovered with the "SQLGetInfo" ODBC function. There are over 150 different discovery attributes to this function. This table has only those points where EDO differs from the official ODBC driver. For a more elaborate description of the "SQLGetInfo" function, please check the Microsoft MSDN site on this subject.

Attribute

Description

Oracle

EDO

SQL_ALTER_TABLE

Support for alter table statement. The oracle drivers erroneously reports that it does not support the following capabilities:

- Add column

- Drop column

- Drop a columns default

- Drop a table constraint

- Set a columns default

SQL_CREATE_TABLE

Support for the create table statement. The oracle drivers erroneously reports that it does not support the following capabilities:

- Deleted rows preserved on commit in a temporary table

- Deleted rows are deleted on commit in a temporary table

- Global temporary tables can be created

SQL_SQL92_FOREIGN_-KEY_DELETE_RULE

Support for the foreign-key relation at the deletion of a row from a detail or master table in the database. The oracle drivers erroneously reports that it does not support the following capabilities:

- Cascaded delete of the details

- No action at the deletion (no foreign keys)

- Default action

- Nullify of the foreign key in the detail table

SQL_DROP_SCHEMA

Oracle does not posses the drop schema statement, as it does not posses a schema concept. But you can cascaded drop a user, and by this drop all the objects of the user, and as such remove the user's schema. The oracle drivers erroneously reports that it does not support the following capabilities:

- Drop the user's schema

- Drop the user's schema with the 'cascade' option

- Drop the user's schema with the 'restrict' option

SQL_DROP_TABLE

Oracle does posses the capability to cascaded or restricted drop a table. The oracle drivers erroneously reports that it does not support the following capabilities:

- Drop the table with the 'cascaded' option

- Drop the table with the 'restricted' option

SQL_SQL92_GRANT

Oracle does posses the grant statement. The oracle drivers erroneously reports that it does not support the following capabilities:

- Grant rights to insert rows in a table

- Grant rights to select rows from a table

- Grant rights to update rows in a table

- Grant rights to delete rows from a table

- Grant rights to reference rows from another table

- Grant rights to insert on special columns in a table

- Grant rights to update on special columns in a table

- Grant rights to reference to special columns in another table

SQL_SQL92_REVOKE

Oracle does posses the revoke statement. The oracle drivers erroneously reports that it does not support the following capabilities:- Grant rights to insert rows in a table

- Revoke rights to select rows from a table

- Revoke rights to update rows in a table

- Revoke rights to delete rows from a table

- Revoke rights to reference rows from another table

- Revoke rights to insert on special columns in a table

- Revoke rights to update on special columns in a table

- Revoke rights to reference to special columns in another table

- Revoke rights in a cascaded way (grantings revoked as well)

- Revoke rights in a restricted way (grantings restrict the revoke action)

- Revoke the "with grant option"

SQL_INDEX_KEYWORDS

Oracle can create it's index in a ascending as well as in a descending way. The oracle drivers erroneously reports that it does not support the following capabilities:

- Creating indexes in descending order

SQL_SQL92_VALUE_-EXPRESSIONS

The oracle drivers erroneously reports that it does not support the following capabilities in a SELECT statement:

- CAST: Casting the value to another datatype

- COALESCE: Function for the "||" operator

- NULLIF: Function to replace a value with a NULL

SQL_OJ_CAPABILITIES

and

SQL_SQL92_RELATIONAL_-JOIN_OPERATORS

In the FROM part of the SELECT statement, tables can be joined together in the select. The oracle drivers erroneously reports that it does not support the following capabilities:

- INNER JOIN

- NATURAL JOIN

- LEFT OUTER JOIN

- RIGHT OUTER JOIN

- FULL OUTER JOIN

- CROSS JOIN

SQL_SQL92_ FOREIGN_KEY-_UPDATE_RULE

Support for the foreign-key relation at the update of a row from a detail or master table in the database. The oracle drivers erroneously reports that it does not support the following capabilities:

- Cascaded delete of the details

- No action at the deletion (no foreign keys)

- Default action

- Nullify of the foreign key in the detail table

SQL_CATALOG_NAME

The oracle drivers erroneously reports that it does not support catalog names. But it does as in "owner.table@catalog" construct. Which is something the official driver does report.

SQL_MAX_CATALOG_-NAME_LEN

The oracle drivers erroneously reports that a catalog name has a length of zero (0) characters. In fact they can be 30 characters long.

SQL_COLLATION_SEQ

The oracle drivers erroneously does not report the name of the collation sequence (character set) for the standard character fields. EDO does report back the collation sequence of the database. e.g.: "WE8MSWIN1252" for a database on a standard European MS-Windows operating system.

SQL_NULL_COLLATION

Every database can have the NULL's fysically stored at another position in a table column. The oracle drivers erroneously reports that it does store nulls in the low end of a column. But in fact it does store them in the high end (after the highest value). EDO does report "SQL_NC_HIGH" here.

SQL_DATETIME_LITERALS

Datetime literals are a way to write dates, times and timestamps in a SQL expression. The oracle drivers erroneously reports that it does not support the following capabilities:

- Dates

- Times

- Timestamps

- Interval year to month

- Interval day to second

SQL_INTEGRITY

The oracle drivers erroneously reports that it does not support transaction integrity!!
