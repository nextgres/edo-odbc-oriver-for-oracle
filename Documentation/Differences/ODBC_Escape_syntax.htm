ODBC Escape syntax

Despite the ISO:9075 SQL standard, not all SQL RDBMS's have the same syntax for a lot of things. That makes programming a general application that can connect to more than one database engine a bit harder. To come by the hardships of all the different syntaxes in the SQL world, the ODBC standard has an escape syntax. This makes it possible to write SQL constructs that an ODBC driver can translate to it's local slang.

Below are various escape syntaxes and how Oracle and EDO supports them:

What

Description

Oracle

EDO

{ } Escapes

Start and end delimiters of the escape sequence

SELECT { fn USER() } FROM dual;

--(* *)-- Escapes

Start and end delimiters of the escape sequence for keyboards without an '{' and '}'

SELECT --(* fn USER() *)-- FROM dual;

--(* with "Vendor"

Start and end delimiters of the escape sequence for keyboards without an '{' and '}' with the VENDOR / Product   included

SELECT --(* VENDOR(Microsoft), PRODUCT(odbc) fn USER() *)-- FROM dual;

Empty escape

Escape sequence with no special other function.

SELECT { * } from dual;

Standard function

Calling standardized functions in a portable way across databases.

SELECT { fn ABS(field)} FROM table

Partly

Procedure call

Way to call a stored procedure (PSM) in the database

{ call mytest(a,b)};

Function call within syntax

Way to call a stored function (PSM) within another SQL construct

SELECT { myfunc()} FROM dual;

Function call

Way to call a stored function (PSM) in one statement only

{ ? = myfunc(a,b) }

Date literal

Specifying a date literal in a SQL statement

SELECT { d'2008-07-30'} FROM dual;

Time literal

Specifying a time literal in a SQL statement

SELECT { t'12:34:47'}    FROM dual;

Timestamp literal

Specifying a timestamp literal in a SQL statement

SELECT { ts'1991-08-16 22:55:00'} AS remi FROM dual;

Interval

Specifying a interval literal in a SQL statement

SELECT { interval '172:45:20' HOUR(3) TO SECOND } FROM DUAL

Escape

Specifying an escape character for the LIKE operator

SELECT field FROM table WHERE field LIKE 'A\_B' { ESCAPE '\' }

Outer join syntax

Way to specify an outer join in a table select statement

SELECT m.id, d.master

FROM { oj master m LEFT OUTER JOIN detail d ON d.master = m.id }

More than one outer join

Way to specify more than one outer join in a table select statement

SELECT m.id, d.master, n.field

   FROM { oj master m   LEFT OUTER JOIN detail d ON d.master = m.id }

       ,{ oj master m2 LEFT OUTER JOIN next n    ON n.master = m.id }

  WHERE m.id = m2.id
