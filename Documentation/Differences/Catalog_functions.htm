Catalog functions

Applications can discover database objects by way of the socalled catalog functions of the ODBC standard. These functions are:

Function

Description

SQLTables

Discover the existence of tables in the database

SQLColumns

Discover the existence of columns in a table

SQLPrimaryKeys

Discover the existence of primary keys in tables

SQLStatistics

Discover the existence of indexes in tables

SQLSpecialColumns

Discover the existence of special automatic and/or hidden columns like ROWID or ROWNUM

SQLProcedures

Discover the existence of stored procedures and/or functions in the database

SQLTablePriveliges

Discover the granted rights to users on tables in the database

SQLColumnPriveleges

Discover the granted rights to users on columns of tables in the database

 

All functions take two strings as the first input parameters (CATALOG and SCHEMA). Schema can here for Oracle be translated as user. The following differences exist between the EDO implementation and the official Oracle ODBC driver:

  • SQLTables can take "SQL_ALL_CATALOGS" as input for a catalog name. This way you can discover all catalogs in the database. EDO implements this option to discover all the database links in a database as openings to other catalogs;

  • SQLTables can take "SQL_ALL_SCHEMAS" as input for a schema name. This way you can discover all schema's in the database. The official Oracle driver returns the users that have defined database objects. EDO returns you all the users, even if the have not yet defined database objects;

  • SQLTables can take "SQL_ALL_TABLE_TYPES" as input for a table type. This retrieves a list of all object types. EDO returns one more table type than the official Oracle driver: "SYNONYM";

  • SQLStatistics has an extra parameter called "reserved". This parameter can take a value "SQL_ENSURE" so that the function will only return after it has 'ensured' itself of the correct statistics. In effect, this means for Oracle that an 'analyze table' has to be done on the table for which the statistics are asked. The official Oracle ODBC driver will never do that. In EDO you can turn the "Ensure statistics info" option on in the ODBC settings of the datasource, or with an extra connection attribute. After you have turned on this option, the SQLStatistics function will work as described in the ODBC standard.
