Database naming

Applications occasionally must know the name of the database they are connecting to. The standard way to retrieve this name in ODBC is to use the "SQLGetInfo" function with the "SQL_DATABASE_NAME" attribute.

The official Oracle ODBC driver strangely returns an empty string in all versions. An alternative method would be to use the "DATABASE" function. Not an entirely portable way to retrieve the database name because we must select it from something this way. But that method also fails, as the "{ fn DATABASE() }" function returns an empty string upto Oracle 11g. That is the first version where this function does not return the "HY00C" error message (Optional feature not implemented).

Of course EDO implements this attribute of the "SQLGetInfo" function in the correct way.
