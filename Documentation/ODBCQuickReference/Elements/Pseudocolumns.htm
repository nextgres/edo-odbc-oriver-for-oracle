Pseudocolumns

Pseudocolumns are similar to table columns. You can query any of these pseudocolumns, but you cannot change their values with DML statements. Which pseudo columns are used is highly dependent on the RDBMS you are using. Here are some well known pseudo's:

Column name

Value

sequence.CURRVAL

ORACLE: Current value of sequence for the current session (sequence.NEXTVAL must be referenced first.)

sequence.NEXTVAL

ORACLE: Next value of sequence for the current session

[table.]LEVEL

ORACLE: 1 for a root node, 2 for a child of a root, and so on. Used in the SELECT statement during tree-structured queries.

[table.]ROWID

ORACLE: Value that uniquely identifies a single row among other rows in the table. ROWID values are of datatype ROWID, not NUMBER or CHAR.

[table.]ROWID

INFORMIX: Integer value that uniquely indentifies a single row among other rows in a table segement on a per dbspace basis.

ROWNUM

ORACLE: Position of a single row among other rows selected by a query. Oracle server selects rows in an arbitrary order and evaluates ROWNUM before sorting rows for an ORDER BY clause.
